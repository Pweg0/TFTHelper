  <name>Board State Parser</name>
  <files>src/main/game/types.ts, src/main/overlay/BoardStateParser.ts, src/main/overlay/BoardStateParser.test.ts</files>
  <behavior>
    - parseBoardState(data: LiveClientResponse) returns DisplayPlayer[] sorted by HP desc
    - Players with isDead === true are excluded from output
    - Players with championStats.currentHealth <= 0 are excluded from output
    - Missing championStats falls back to hp: 0 (which means excluded) — use "?" sentinel only in display layer
    - Each DisplayPlayer contains: summonerName, hp, maxHp, level, champions (string[]), items (per-champion), isLocalPlayer
    - Local player is identified by matching activePlayer.summonerName against allPlayers[n].summonerName
    - Local player includes gold from activePlayer.currentGold
    - Zod LiveClientResponseSchema validates with .passthrough() and .optional() for uncertain TFT fields
    - Empty allPlayers array returns empty DisplayPlayer[]
    - Duplicate summonerNames are preserved (no dedup)

    Test cases:
    - 8 players with varying HP -> returns sorted by HP desc, eliminated filtered out
    - All players eliminated -> returns empty array
    - Player missing championStats -> treated as eliminated (filtered out)
    - activePlayer.summonerName matches one player -> that player has isLocalPlayer=true and gold
    - Empty allPlayers -> returns []
    - Zod schema accepts extra unknown fields without throwing
  </behavior>
  <implementation>
    1. Expand types.ts: Add TFTPlayer, TFTItem, TFTChampion interfaces with index signatures for passthrough. Update LiveClientResponse.allPlayers from unknown[] to TFTPlayer[]. Add ActivePlayer interface with currentGold, level, summonerName. Add DisplayPlayer and DisplayChampion interfaces.

    2. Create BoardStateParser.ts with:
       - Zod schemas (TFTItemSchema, TFTPlayerSchema, ActivePlayerSchema, LiveClientResponseSchema) using .passthrough() and .optional() on uncertain fields
       - parseBoardState(data: LiveClientResponse): DisplayPlayer[] function
       - Filter: exclude isDead===true or currentHealth<=0
       - Sort: HP descending
       - Map: extract summonerName, hp, maxHp, level, champions list, items per champion
       - Local player: match activePlayer.summonerName, attach gold

    NOTE on TFT Live Client API structure (from research): The allPlayers array in TFT may represent individual champion units rather than players. The exact field names are provisional (LOW confidence from research). Use .passthrough() and .optional() everywhere. The real schema will be validated with a live game in Plan 04. Build the parser to handle both interpretations gracefully.
  </implementation>
