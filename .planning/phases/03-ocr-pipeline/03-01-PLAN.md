    - ScreenCapturer reads actual thumbnail size from .getSize() to detect DPI scaling
    - cropRegion(pngBuffer, x, y, w, h) returns a preprocessed PNG Buffer (cropped, grayscaled, thresholded, 3x upscaled)
    - cropRegion with out-of-bounds coordinates throws or clamps gracefully

  <action>
1. Create `src/main/ocr/ScreenCapturer.ts`:
   - Class with `async capture(windowTitle: string): Promise<{ png: Buffer; width: number; height: number } | null>`
   - Uses `desktopCapturer.getSources({ types: ['window'], thumbnailSize: { width: 1920, height: 1080 } })`
   - Finds source by `s.name === windowTitle` (exact match, with fallback `.includes('League of Legends')`)
   - Returns `{ png: source.thumbnail.toPNG(), width: size.width, height: size.height }`
   - Returns null if no matching source found

   - Export `async function cropRegion(pngBuffer: Buffer, x: number, y: number, w: number, h: number): Promise<Buffer>`
   - Uses Jimp: `Jimp.fromBuffer(pngBuffer)` then `.crop({x, y, w, h}).greyscale().threshold({max: 128}).scale(3)`
   - Returns `cropped.getBuffer('image/png')`
