<task type="auto" tdd="true">
  <name>Task 1: BoardOCR for bench and board champion recognition with item detection</name>
  <files>src/main/ocr/BoardOCR.ts, src/main/ocr/BoardOCR.test.ts</files>
  <behavior>
    - readBench(pngBuffer, width, height, matcher) returns OCRChampion[] for 9 bench slots
    - readBoard(pngBuffer, width, height, matcher) returns OCRChampion[] for occupied board slots
    - Empty bench/board slots return empty arrays (not nulls)
    - Uses text OCR on name labels below champions (per RESEARCH.md Pitfall 5 recommendation: text OCR safer than pixel template matching for board)
    - Star level defaults to 1 (placeholder -- accurate star detection is best-effort)
    - readItems(pngBuffer, slotCenter, width, height) returns string[] of matched item apiNames by pixelmatch against cached item icons on 3 item slot positions at champion corners (~24x24px regions)
    - Unrecognized items (mismatch count above threshold) are excluded from the array
  </behavior>
  <action>
1. Create `src/main/ocr/BoardOCR.ts`:
   - Class that shares the Tesseract worker from ShopOCR (pass worker reference in constructor, or accept a worker factory)
   - Simplest approach: BoardOCR constructor takes the same Tesseract worker used by ShopOCR (single worker, passed from OCRPipeline).
   - Constructor also accepts an item icon cache: `Map<string, Buffer>` mapping item apiName to 24x24 RGBA pixel buffer (loaded from Phase 1 cached item icon PNGs at startup)
   - `async loadItemIconCache(itemIconPaths: Map<string, string>): Promise<void>`
     - For each item, load the cached PNG via Jimp, resize to 24x24, extract raw RGBA bitmap data
     - Store in `this.itemIconCache: Map<string, Buffer>` (apiName -> 24x24 RGBA)
   - `async readItems(pngBuffer: Buffer, slotCenter: {x: number, y: number}, width: number, height: number): Promise<string[]>`
     - TFT renders up to 3 item icons at the bottom-left, bottom-center, and bottom-right of each champion cell
     - Item slot offsets relative to slot center: approximately [{x: -20, y: +20}, {x: 0, y: +20}, {x: +20, y: +20}] (bottom edge of champion cell, ~24x24 each)
     - For each of 3 item slot positions: crop the 24x24 region from the screenshot, get raw RGBA via Jimp
     - Use pixelmatch to compare against every icon in itemIconCache. Best match below a mismatch threshold (e.g., mismatch count < 150 out of 24*24=576 pixels) is a valid item
     - Return array of matched item apiNames (may be empty, 1, 2, or 3 items)
     - If no item matches threshold, the slot is empty -- skip it
   - `async readBench(pngBuffer: Buffer, width: number, height: number, matcher: ChampionMatcher): Promise<OCRChampion[]>`
     - For each of 9 BENCH_SLOTS: compute scaled coordinates, crop a name label region below the slot center (~60x15px band below center), run Tesseract recognize, fuzzy match via matcher
     - For recognized champions: also call readItems() to detect items at that slot
     - Filter out unrecognized slots (where matcher returns null AND confidence < 40, indicating empty slot)
     - Return array of OCRChampion with starLevel: 1 and itemApiNames from readItems()
   - `async readBoard(pngBuffer: Buffer, width: number, height: number, matcher: ChampionMatcher): Promise<OCRChampion[]>`
     - Same approach but for BOARD_SLOTS (28 positions)
     - Most slots will be empty -- filter out low-confidence empty reads
     - For recognized champions: call readItems() to detect items

2. Create test: mock Tesseract worker, verify readBench processes 9 slots, verify readBoard processes 28 slots, verify empty results when worker returns garbage. Mock itemIconCache with a synthetic 24x24 buffer. Verify readItems returns matching apiName when pixelmatch finds a match below threshold. Use mock ChampionMatcher.

  <action>
1. Create `src/main/ocr/OCRPipeline.ts`:
   - Class that owns: ScreenCapturer, ShopOCR, BoardOCR, ChampionMatcher instances
   - `async initialize(champions: Champion[], items: Item[]): Promise<void>` -- creates ChampionMatcher from champion list, initializes ShopOCR Tesseract worker, shares worker with BoardOCR, loads item icon cache into BoardOCR from item icon paths (items have .icon path to cached PNG from Phase 1)
   - `async tick(): Promise<OCRResult>` -- single-tick orchestration:
     a. Capture screenshot via ScreenCapturer (window title from constant)
     b. If capture returns null, return offline result
     c. Check shop visibility via isShopVisible
     d. If shop visible: run ShopOCR.readShop
     e. Run BoardOCR.readBench and BoardOCR.readBoard (these now include item detection)
     f. Compute owned flag on shop slots: for each ShopSlot, set owned = true if apiName matches any champion in board or bench arrays
     g. Return OCRResult with ocrStatus: 'active'
   - `async terminate(): Promise<void>` -- cleanup Tesseract worker
   - Stale data tracking: store `lastValidResult` and `lastValidAt` timestamp
     - If tick() capture returns null: check if (Date.now() - lastValidAt) < 10000 -> return lastValidResult with ocrStatus: 'stale'
     - If > 10 seconds of null: return empty result with ocrStatus: 'offline'

2. Update `src/main/overlay/BoardStatePoller.ts`:
   - Add OCRPipeline as a dependency (create in constructor or via setter)
   - In `start()`: call `ocrPipeline.initialize(champions, items)` (champions and items passed as parameter or loaded via IPC)
   - In `tick()`: call both `fetchGameData()` AND `ocrPipeline.tick()` in parallel
   - Merge results: parseOverlayState for API data + OCR result fields
   - Send merged OverlayState via IPC

3. Update `src/main/overlay/BoardStateParser.ts`:
   - Update `parseOverlayState` to accept optional OCRResult parameter
   - When OCRResult provided, merge its fields into the OverlayState
   - When OCRResult not provided, use defaults: board: [], bench: [], shop: [], shopVisible: false, ocrStatus: 'offline'
